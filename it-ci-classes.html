CI Class & Cấu trúc phân cấp trong CMDBCI Classes & CMDB Class HierarchyCIクラスとCMDB階層構造

Hiểu cách ServiceNow tổ chức CI theo lớp kế thừa — nền tảng để thiết kế CMDB đúng cách.Understand how ServiceNow organizes CIs in an inheritance hierarchy — the foundation of proper CMDB design.ServiceNowが継承階層でCIを整理する仕組みを理解する——正しいCMDB設計の基礎。

📅 2025.10⏱ 8 min✍ Ain (Nguyen Anh)

Trong ServiceNow, mọi CI đều thuộc về một class (lớp). Các class được tổ chức theo cấu trúc kế thừa — giống như cây phân loại sinh vật — từ tổng quát đến cụ thể.

In ServiceNow, every CI belongs to a class. Classes are organized in an inheritance hierarchy — like a taxonomy tree — from general to specific.

ServiceNowでは、すべてのCIはクラスに属しています。クラスは継承階層で整理されており——生物の分類のように——一般的なものから具体的なものへと構成されています。

🎯 Key Exam Point

Tất cả CI class đều kế thừa từ cmdb_ci — đây là base table của toàn bộ CMDB. Child class tự động kế thừa tất cả attributes của parent class.

All CI classes inherit from cmdb_ci — the base table of the entire CMDB. Child classes automatically inherit all attributes from parent classes.

すべてのCIクラスはcmdb_ci（CMDBのベーステーブル）から継承します。子クラスは親クラスのすべての属性を自動的に継承します。

03 Principal Class là gì?What is Principal Class?プリンシパルクラスとは？

Principal Class là các class được đánh dấu là "quan trọng" và sẽ xuất hiện trong CI list views mặc định. Khi bạn bật Principal Class filter, chỉ CI thuộc các class này mới hiển thị — giúp tránh nhiễu từ hàng trăm class con.

Principal Classes are classes marked as "important" that appear in default CI list views. When you enable the Principal Class filter, only CIs from these classes are shown — avoiding noise from hundreds of subclasses.

プリンシパルクラスは「重要」とマークされ、デフォルトのCIリストビューに表示されるクラスです。プリンシパルクラスフィルターを有効にすると、これらのクラスのCIのみが表示されます——数百のサブクラスによるノイズを防ぎます。

04 Reclassify CIReclassifying a CICIの再分類

Đôi khi bạn cần đổi class của một CI (ví dụ từ cmdb_ci_computer lên cmdb_ci_server). Quá trình này gọi là reclassification:

Sometimes you need to change a CI's class (e.g., from cmdb_ci_computer to cmdb_ci_server). This is called reclassification:

時にCIのクラスを変更する必要があります（例：cmdb_ci_computerからcmdb_ci_serverへ）。これを再分類と呼びます：

TypeDescriptionExample
UpgradeĐổi sang class con (thêm attributes)Move to child class (gain attributes)子クラスへ移行（属性追加）computer → server
DowngradeĐổi sang class cha (mất attributes)Move to parent class (lose attributes)親クラスへ移行（属性消失）server → computer
SwitchĐổi sang class cùng cấp (nhánh khác)Move to sibling class (different branch)同レベルの異なるクラスへ移行server → router
