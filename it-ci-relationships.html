CI Relationships — Tại sao mối quan hệ giữa CI quan trọng?CI Relationships — Why connections matterCIリレーションシップ — なぜ繋がりが重要か

CMDB không chỉ là danh sách tài sản — mối quan hệ giữa CI mới là thứ biến nó thành công cụ ra quyết định.CMDB is more than an asset list — CI relationships are what turn it into a powerful decision tool.CMDBは資産リストではない——CIのつながりこそが強力な意思決定ツールにする鍵。

📅 2025.11⏱ 9 min✍ Ain (Nguyen Anh)

Nếu CMDB chỉ lưu danh sách CI, nó không khác gì spreadsheet. Điều biến CMDB thành công cụ ra quyết định mạnh mẽ chính là dữ liệu về mối quan hệ giữa các CI.

If CMDB only stored a list of CIs, it wouldn't be different from a spreadsheet. What makes CMDB a powerful decision tool is the relationship data between CIs.

CMDBがCIのリストを格納するだけなら、スプレッドシートと変わりません。CMDBを強力な意思決定ツールにするのは、CI間のリレーションシップデータです。

🎯 Key Exam Point

Mỗi relationship gồm 3 thành phần: Parent CI + Relationship Type + Child CI. Ví dụ: [Server1] [Managed by] [Server2].

Each relationship consists of 3 components: Parent CI + Relationship Type + Child CI. Example: [Server1] [Managed by] [Server2].

各リレーションシップは3つの要素で構成：親CI + 関係タイプ + 子CI。例：[Server1] [Managed by] [Server2]。

01 Cấu trúc của một CI RelationshipStructure of a CI RelationshipCIリレーションシップの構造

Relationship Structure
Parent CI ──[Relationship Type]──▶ Child CI
Example: [Database Server] [Hosts] [Oracle DB Instance]

02 Các loại Relationship phổ biếnCommon Relationship Typesよく使われるリレーションタイプ

Relationship TypeMeaningExample
Hosted onCI chạy trên CI khácCI runs on another CI別CIの上で動作App → Server
Managed byCI được quản lý bởi CI/User khácCI is managed by another CI/User別のCI/ユーザーが管理Server → Admin Group
Depends onCI phụ thuộc vào CI khácCI depends on another CI別のCIに依存App → Database
UsesCI sử dụng tài nguyên của CI khácCI uses resources from another CI別CIのリソースを使用Service → Network Switch
Connects toKết nối vật lý hoặc logicPhysical or logical connection物理的または論理的な接続Router → Switch

03 Dependency Views — Nhìn hạ tầng trực quanDependency Views — Visual Infrastructure依存関係ビュー — インフラの可視化

ServiceNow cung cấp Dependency Views — một công cụ trực quan hóa mối quan hệ giữa CI. Bạn có thể:

ServiceNow provides Dependency Views — a tool to visualize CI relationships. You can:

ServiceNowは依存関係ビューを提供します——CIリレーションシップを視覚化するツールです。以下が可能です：

  • Xem toàn bộ hạ tầng dưới dạng đồ thịView entire infrastructure as a graphインフラ全体をグラフで表示
  • Xem real-time status của từng CIView real-time status of each CI各CIのリアルタイムステータスを確認
  • Truy cập thông tin về incidents, changes, alerts liên quanAccess related incidents, changes, and alerts関連するインシデント・変更・アラートにアクセス
💡 Practical Example

Router bị lỗi → Dependency View hiển thị ngay tất cả Server và Application bị ảnh hưởng → Incident được tự động tạo cho từng CI liên quan.

Router fails → Dependency View instantly shows all affected Servers and Applications → Incidents automatically created for each affected CI.

ルーターが障害 → 依存関係ビューが影響を受けるサーバーとアプリを即座に表示 → 関連するCIごとにインシデントが自動作成。

04 Suggested RelationshipsSuggested Relationships推奨リレーションシップ

ServiceNow cho phép định nghĩa Suggested Relationships cho từng CI class — những mối quan hệ "thường gặp" khi tạo CI thuộc class đó. Điều này giúp:

ServiceNow allows defining Suggested Relationships for each CI class — "typical" relationships when creating CIs of that class. This helps:

ServiceNowでは各CIクラスに推奨リレーションシップを定義できます——そのクラスのCIを作成する際の「典型的な」関係です。これにより：

  • Hướng dẫn người dùng tạo relationship đúngGuide users to create correct relationships正しいリレーション作成をユーザーに案内
  • Đảm bảo tính nhất quán dữ liệu CMDBEnsure CMDB data consistencyCMDBデータの一貫性を確保
